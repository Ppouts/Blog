Projet : Page Web de brut force de mots de passe

1. Présentation du Projet

Client : Gendarmerie Nationale

Nom du projet : Développement d'un outil de brut force

Responsable du projet : POUTS Pierre

2. Objectifs du Projet

  • Créer un outil web pour trouver les mots de passe des fichiers ZIP et PDF
  • Identifier les mots de passe pour accéder aux données
  • Assurer que l'outil soit simple d’utilisation et efficace

3. Description des Fonctionnalités

3.1. Interface Utilisateur

  • Drag and Drop : Pour glisser facilement les fichiers dans la page web
  • Recherche par fichier : Une recherche directe dans les fichiers de l’ordinateur/Clés USB pour une utilisation pratique
  • Visuel : Une page simple avec très peu d’informations

3.2. Fonctionnalités Techniques

  • Module de test de mots de passe : Algorithmes de brute force et dictionnaires pour trouver les mots de passe.
  • Affichage du mot de passe : Renvoie du mot de passe sur la page

4. Contraintes Techniques

  • Technologies : Utilisation de Python et JS pour le développement de l'algorithme de brute force, et Flask pour le front-end.
  • Hébergement : Utilisation d'un serveur sécurisé et hébergé en France

5. Conformité et Éthique

  • Conformité légale : L'outil doit respecter les lois françaises et européennes en matière de cybersécurité et de protection des données.
  • Accès restreint : L'accès à l'outil doit être limité aux utilisateurs autorisés de la gendarmerie.

6. Délai et Planning

  • Durée estimée du projet : 6 semaines
  • Phases du projet :
    • Développement de l’interface utilisateur : 1 semaine
    • Développement de l'algorithme : 4 semaines
    • Tests et validation : 2 jours
    • Tâches annexes : 3 jours
